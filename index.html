1. Γενικός Σκοπός της Πτυχιακής

Σκοπός της παρούσας πτυχιακής εργασίας είναι η ανάπτυξη ενός υβριδικού συστήματος ανίχνευσης επιθέσεων NFC relay σε ανέπαφα συστήματα πληρωμών, όπως το Apple Pay και το Google Pay. Το σύστημα συνδυάζει rule-based μεθόδους με τεχνικές Τεχνητής Νοημοσύνης για να εντοπίζει ύποπτες συναλλαγές σε πραγματικό χρόνο — πριν αυτές εγκριθούν και εκτελεστούν.

Η αρχιτεκτονική αποτελείται από πέντε επίπεδα: συλλογή δεδομένων, rule-based engine, AI detection engine, risk scoring module και επίπεδο απόφασης. Κάθε συναλλαγή αξιολογείται σε milliseconds και λαμβάνει ένα ενοποιημένο risk score που καθορίζει αν θα επιτραπεί, επισημανθεί ή αποκλειστεί.

"Πώς μπορούμε να εντοπίσουμε αυτόματα επιθέσεις NFC relay σε ανέπαφα συστήματα πληρωμών, χρησιμοποιώντας συνδυασμό κανόνων και ML, χωρίς να επιβαρύνουμε την εμπειρία του χρήστη;"

Σύνδεση με Μαθήματα Σπουδών

  • Ασφάλεια Δικτύων & Επικοινωνιών — NFC πρωτόκολλα, threat modeling, relay attack mechanics
  • Μηχανική Μάθηση — Isolation Forest, Random Forest για anomaly detection και classification
  • Συστήματα Πληρωμών — EMV πρωτόκολλο, contactless payment flow, transaction latency
  • Ανάλυση Δεδομένων — Συνθετική παραγωγή dataset, feature engineering, αξιολόγηση μοντέλων

2. Πρόβλημα & Threat Model

Τα ανέπαφα συστήματα πληρωμών χρησιμοποιούν NFC (Near Field Communication) για γρήγορες συναλλαγές σε απόσταση έως 4cm. Ωστόσο, μια επίθεση NFC relay επιτρέπει σε έναν επιτιθέμενο να αναμεταδίδει την επικοινωνία μεταξύ μιας νόμιμης συσκευής και ενός τερματικού πληρωμών, εξαπατώντας το σύστημα ότι η κάρτα βρίσκεται κοντά, ενώ στην πραγματικότητα μπορεί να είναι χιλιόμετρα μακριά.

⚠ ΠΑΡΑΔΕΙΓΜΑ ΕΠΙΘΕΣΗΣ
Ο επιτιθέμενος A στέκεται κοντά στο θύμα (π.χ. σε μετρό). Ο επιτιθέμενος B στέκεται μπροστά σε ένα POS terminal. Η επικοινωνία μεταξύ κάρτας θύματος και POS αναμεταδίδεται wirelessly — η συναλλαγή εγκρίνεται χωρίς η κάρτα να είναι ποτέ κοντά στο terminal.

Γιατί τα Υπάρχοντα Συστήματα Αποτυγχάνουν

  • Rule-based μόνο: Εντοπίζουν γνωστά patterns αλλά αδυνατούν να ανιχνεύσουν εξελιγμένες ή νέες παραλλαγές επιθέσεων
  • AI-only μόνο: Ανιχνεύουν λεπτές ανωμαλίες αλλά με υψηλό false positive rate — αποκλείουν νόμιμες συναλλαγές
  • Έλλειψη real-time ανάλυσης: Τα περισσότερα συστήματα ανιχνεύουν απάτη μετά την ολοκλήρωση της συναλλαγής

Το σύστημα αποτελείται από πέντε επίπεδα που επεξεργάζονται κάθε συναλλαγή σε πραγματικό χρόνο:

Επίπεδο 1 — Data Collection Layer

Συλλέγει σε πραγματικό χρόνο τα χαρακτηριστικά κάθε συναλλαγής που είναι απαραίτητα για την ανίχνευση relay attacks:

  • Transaction Latency — καθυστέρηση round-trip σε milliseconds (relay attacks εισάγουν επιπλέον καθυστέρηση)
  • Transaction Amount — ποσό συναλλαγής (ασυνήθιστα μεγάλα ποσά σε νέο terminal)
  • Transaction Frequency — αριθμός συναλλαγών ανά χρονικό παράθυρο
  • Device State — κατάσταση συσκευής (screen on/off, app active, location services)
  • Contextual Info — γεωγραφική τοποθεσία, ώρα ημέρας, merchant category

Επίπεδο 2 — Rule-Based Engine

Ελέγχει για γνωστά, προκαθορισμένα patterns που υποδηλώνουν relay attack:

# Παράδειγμα κανόνων:
IF latency > 300ms          → SUSPICIOUS  (relay adds ~100-500ms delay)
IF transactions > 3/minute  → SUSPICIOUS  (rapid repeated attempts)
IF amount > €500 AND
   new_merchant = True       → SUSPICIOUS  (high value, unknown terminal)
IF device_screen = OFF AND
   transaction_initiated     → SUSPICIOUS  (card used without user action)

Επίπεδο 3 — AI Detection Engine

Δύο συμπληρωματικά ML μοντέλα για ανίχνευση ανωμαλιών και ταξινόμηση:

  • Isolation Forest — unsupervised anomaly detection, εντοπίζει στατιστικά ασυνήθιστες συναλλαγές χωρίς labeled data
  • Random Forest Classifier — supervised classification (legitimate / relay attack) με explainability μέσω feature importance
# AI Model Output:
{
  "isolation_forest_score": -0.42,   (negative = anomaly)
  "rf_prediction":          "relay_attack",
  "rf_confidence":          0.89,
  "top_features": [
    "latency_ms: 387 (threshold: 150)",
    "tx_frequency: 5 in 60s",
    "device_state: screen_off"
  ]
}

Επίπεδο 4 — Risk Scoring Module

Συνδυάζει τα αποτελέσματα των δύο engines σε ένα ενοποιημένο risk score (0–100) χρησιμοποιώντας weighted aggregation. Ο συνδυασμός μειώνει τα false positives του AI και τα blind spots των κανόνων.

Επίπεδο 5 — Decision Layer

  • Score < 30: ALLOW — η συναλλαγή προχωράει κανονικά
  • Score 30–70: FLAG — η συναλλαγή επισημαίνεται για manual review ή επιπλέον authentication
  • Score > 70: BLOCK — η συναλλαγή αποκλείεται αυτόματα και αποστέλλεται alert

6. ML Μοντέλα — Σημαντικές Λεπτομέρειες

Isolation Forest — Anomaly Detection

Ο Isolation Forest εντοπίζει anomalies χωρίς labeled data (unsupervised). Χτίζει τυχαία decision trees και μετρά πόσα splits χρειάζονται για να απομονωθεί ένα sample. Τα relay attacks δημιουργούν μοναδικά patterns (π.χ. ασυνήθιστο latency) που απομονώνονται γρήγορα — άρα λαμβάνουν χαμηλό anomaly score.

  • Ιδανικό για αρχικό φιλτράρισμα χωρίς labeled examples
  • Robust σε class imbalance (relay attacks είναι σπάνια)
  • Γρήγορο inference — κατάλληλο για real-time ανάλυση

Random Forest Classifier — Classification

Εκπαιδεύεται σε labeled συνθετικό dataset για να διακρίνει νόμιμες συναλλαγές από relay attacks. Παρέχει feature importance scores που εξηγούν γιατί μια συναλλαγή θεωρείται ύποπτη.

  • Ensemble μέθοδος — συνδυάζει πολλά decision trees για robustness
  • Παρέχει probability scores αντί binary αποφάσεων
  • Built-in explainability μέσω feature importance (χρήσιμο για audit)

ΘΕΜΑ 2 - Σύνοψη Πρότασης

Το Θέμα 2 εστιάζει στην ανίχνευση και αποφυγή browser fingerprinting με συνδυασμό heuristic τεχνικών και ML. Κεντρική ιδέα είναι η επέκταση των framework Carnus/Chronos/Human Touch/Simulacrum με ένα νέο ML επίπεδο που θα λειτουργεί τόσο για detection όσο και για robust defense.

"Από rule-based signatures σε ML-based γενίκευση: στόχος είναι η ανίχνευση νέων και εξελισσόμενων fingerprinting τεχνικών, με χαμηλό web breakage."

1. Εισαγωγή και Κίνητρο

Το browser fingerprinting είναι stateless τεχνική παρακολούθησης που συνθέτει μοναδικό αναγνωριστικό από στοιχεία όπως canvas, WebGL, fonts, AudioContext, storage και behavioral signals. Με τη σταδιακή μείωση των third-party cookies, οι trackers χρησιμοποιούν περισσότερο fingerprinting, first-party cookie ghost-writing και service worker abuse.

Η ερευνητική γραμμή του Δρ. Ηλία αποτελεί σημείο αναφοράς διεθνώς στο extension fingerprinting και web tracking. Ωστόσο, τα περισσότερα συστήματα παραμένουν heuristic/rule-based, άρα υπάρχει ευκαιρία για ML γενίκευση και ανθεκτικότητα απέναντι σε obfuscation και evasive tactics.

2. Βασικά Ευρήματα από την Υφιστάμενη Βιβλιογραφία

ΕργασίαΚύριο αποτέλεσμαΚενό που μένει
Carnus (NDSS 2020) Μεγάλης κλίμακας extension fingerprinting με WAR/behavior signatures και 29K+ μοναδικά extensions. Exact signature matching, περιορισμένη γενίκευση σε παραλλαγές.
Human Touch (USENIX 2022) Αποδεικνύει ότι user-triggered interactions αποκαλύπτουν επιπλέον extension fingerprints. Χειροκίνητα templates, περιορισμένη κάλυψη σύνθετων interaction chains.
Chronos (CCS 2022) Continuous fingerprinting με MutationObserver streams και σημαντική αύξηση ανίχνευσης. Παραμένει heuristic pipeline, ευαισθησία σε noise/collisions.
Sleeper Agents (NDSS 2021) Service workers μπορούν να χρησιμοποιηθούν για history sniffing και privacy leakage. Manual rules, χωρίς αυτόματη ML διάκριση benign/malicious patterns.
Cookie Swap Party (WWW 2021) First-party cookies από third-party JS χρησιμοποιούνται εκτεταμένα για tracking. Heuristic identifier matching με όρια σε σύνθετα encodings.
Simulacrum (USENIX 2022) DOM reality shifting για αποτροπή extension fingerprinting. Ισχυρή άμυνα αλλά όχι ML-driven adaptive detection.

  • ΕΕ1: Μπορεί ML να ταξινομεί fingerprinting scripts/traces με υψηλή ακρίβεια και robust συμπεριφορά σε obfuscation;
  • ΕΕ2: Μπορεί unsupervised learning να ανακαλύπτει νέα fingerprinting clusters πέρα από γνωστές signatures;
  • ΕΕ3: Πόσο ευάλωτοι είναι οι detectors σε evasive tactics (WASM μεταφορά, perturbations, timing evasion);
  • ΕΕ4: Μπορεί να υλοποιηθεί πρακτικό browser countermeasure με on-device ML inference και χαμηλό breakage;

Συλλογή δεδομένων από Tranco/OpenWPM και extension-focused traces (Carnus/Chronos λογική), με multi-class labels: canvas/webgl/audio/font FP, extension behavioral FP, first-party cookie abuse, service worker abuse, legitimate.

Στάδιο 2 - Feature Engineering

  • Syntactic: AST tokens, API frequencies.
  • Semantic: execution trace sequences με context.
  • Behavioral: MutationObserver records σε χρονική ακολουθία.
  • Graph: script-storage-network information flows.

Στάδιο 3 - Supervised Detection

Σύγκριση Random Forest, XGBoost, 1D-CNN και Transformer/bytecode model με μετρικές Accuracy, Precision, Recall, F1, FPR, και breakage rate.

Στάδιο 4 - Unsupervised Discovery

Clustering και anomaly detection για εντοπισμό unknown fingerprinting patterns που δεν ταιριάζουν σε γνωστές signatures.

Στάδιο 5 - Evasion/Adversarial Study

  • WASM-based μεταφορά fingerprinting λογικής.
  • Feature perturbations (FGSM/PGD) σε embeddings.
  • Mutation/order/timing perturbations για αποφυγή instrumentation.

Στάδιο 6 - Defense Prototype (SpectralShield)

Browser extension με quantized on-device model (ONNX Runtime Web), που ενεργοποιεί Simulacrum-style DOM reality shifting όταν ανιχνεύεται fingerprinting script.

  • Σαφής σύγκριση 3-4 ML αρχιτεκτονικών με κοινό benchmark.
  • Ποσοτικοποίηση πτώσης απόδοσης υπό evasion και βελτίωση με adversarial training.
  • Prototype του SpectralShield με στόχο χαμηλό runtime overhead.
  • Υλικό κατάλληλο για submission σε workshop/venue privacy-security.

ΘΕΜΑ 3 - Σύνοψη Πρότασης

Η πρόταση εστιάζει στην ανίχνευση cryptojacking σε πραγματικό χρόνο μέσα στον browser, συνδυάζοντας behavioral analysis και ML. Χτίζει πάνω στην εργασία του επιβλέποντα "Truth in Web Mining" (ISC 2019), μεταφέροντας τη μελέτη από measurement-only σε πρακτικό detection/defense pipeline.

"Από το πόσο κοστίζει το cryptojacking στον χρήστη, στο πώς το εντοπίζουμε αξιόπιστα και έγκαιρα όταν είναι obfuscated ή WASM-based."

1. Εισαγωγή και Σχέση με SPT@CUT

Το cryptojacking χρησιμοποιεί τους πόρους του θύματος (CPU/GPU, ενέργεια, θερμοκρασία, απόκριση συστήματος) για εξόρυξη κρυπτονομισμάτων χωρίς συγκατάθεση. Η προηγούμενη γραμμή έρευνας του Δρ. Ηλία ποσοτικοποίησε το overhead και την κερδοφορία του φαινομένου, με βασικό εύρημα ότι η επίθεση γίνεται κερδοφόρα όταν ο χρήστης παραμένει πάνω από 5.53 λεπτά στο site.

Το Θέμα 3 επεκτείνει αυτή τη βάση σε σύγχρονο πλαίσιο 2025+, όπου η δραστηριότητα μετατοπίζεται προς obfuscated scripts, WASM payloads και supply-chain injection σε third-party resources.

2. Το Πρόβλημα Σήμερα

  • Obfuscation gap: σημαντικό ποσοστό cryptojacking scripts είναι obfuscated, άρα τα blacklist/AV μοντέλα αποτυγχάνουν συχνά.
  • WASM gap: mining μέσω WebAssembly είναι δυσκολότερο να εντοπιστεί με κλασική static ανάλυση JS.
  • Generalization gap: rule-based μηχανισμοί δεν γενικεύουν σε νέες παραλλαγές και evasive στρατηγικές.
  • Behavioral gap: πολλά υπάρχοντα ML μοντέλα βασίζονται σε λίγα coarse features και όχι σε browser API-level behavior.

Στάδιο 1 - Measurement Study

Automated crawl σε Tranco Top-5000 με OpenWPM και συλλογή μετρήσεων: CPU usage ανά tab, ενεργά WebWorkers, κλήσεις `WebAssembly.instantiate()`, outbound requests προς mining pools και πιθανό throttling behavior.

Παραδοτέο: ενημερωμένο labeled dataset (`cryptojacking` / `legitimate`) που επεκτείνει πρακτικά το measurement scope του 2019.

Στάδιο 2 - ML Detection Model

Ομάδα FeaturesΧαρακτηριστικάΣτόχος
Runtime Behavioral CPU spike/sustain patterns, πλήθος WebWorkers, memory allocation trends, traffic προς mining pools Ανίχνευση ενεργού mining συμπεριφοράς
Static Script/WASM AST API sequences, call frequencies, WASM signatures/sections Αναγνώριση κακόβουλων μοτίβων πριν ή κατά την εκτέλεση

Μοντέλα προς σύγκριση: Random Forest, XGBoost και lightweight Neural Network. Στόχος απόδοσης: >97% accuracy και <2% false positives.

Στάδιο 3 - Browser Extension Proof of Concept

Υλοποίηση Chrome/Firefox extension που τρέχει inference client-side σχεδόν σε πραγματικό χρόνο. Το extension κάνει hook σε WASM instantiation events και συνδυάζει timing-based indicators για runtime behavior tracking.

  • Alert όταν εντοπιστεί ύποπτο cryptojacking μοτίβο.
  • Επιλογή χρήστη: block script ή επιτρεπτό opt-in mining.
  • Demo-able εργαλείο για τελική παρουσίαση.

4. Καινοτομία σε σχέση με τη Γραμμή 2019

  1. Μετάβαση από profitability/overhead measurement σε real-time ML detection.
  2. Ενσωμάτωση behavioral + static/WASM χαρακτηριστικών σε ενιαίο pipeline.
  3. Εστίαση σε obfuscated και WASM-based miners που διαφεύγουν από rule-based λύσεις.
  4. Παραγωγή browser extension PoC με άμεσο πρακτικό αντίκτυπο για τον χρήστη.

8. Τι Κερδίζει ο Χρήστης

  • Προστασία χωρίς να κάνει τίποτα: το extension εντοπίζει αυτόματα cryptojacking και ενημερώνει τον χρήστη χωρίς να απαιτεί τεχνικές γνώσεις (WASM, mining pools, obfuscation).
  • Προστασία της συσκευής: μειώνεται ο κίνδυνος από sustained υψηλό CPU load που φθείρει μπαταρία/hardware και αυξάνει overheating, ειδικά σε mobile συσκευές και laptops.
  • Οικονομική προστασία: περιορίζεται το κρυφό κόστος από κατανάλωση ενέργειας και data traffic που προκαλεί ο miner του επιτιθέμενου.

9. Τι Κερδίζει η Επιστήμη / η Κοινότητα

  • Updated measurement study: νέα εικόνα του cryptojacking οικοσυστήματος μετά το 2019 (post-Coinhive εποχή, WASM adoption, νέα οικονομικά δεδομένα), με δημόσια αξιοποιήσιμο dataset.
  • Κάλυψη του WASM gap: ML detector που στοχεύει ειδικά WASM-based miners, κενό που δεν καλύπτεται επαρκώς από υπάρχοντα εργαλεία.
  • Επαναχρησιμοποιήσιμη συνεισφορά: μοντέλο και μεθοδολογία που μπορούν να αξιοποιηθούν από άλλους researchers ή να ενσωματωθούν σε πρακτικές λύσεις filtering/blocking.
