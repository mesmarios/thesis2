1. Γενικός Σκοπός της Πτυχιακής

Σκοπός της παρούσας πτυχιακής εργασίας είναι η ανάπτυξη ενός υβριδικού συστήματος ανίχνευσης επιθέσεων NFC relay σε ανέπαφα συστήματα πληρωμών, όπως το Apple Pay και το Google Pay. Το σύστημα συνδυάζει rule-based μεθόδους με τεχνικές Τεχνητής Νοημοσύνης για να εντοπίζει ύποπτες συναλλαγές σε πραγματικό χρόνο — πριν αυτές εγκριθούν και εκτελεστούν.

Η αρχιτεκτονική αποτελείται από πέντε επίπεδα: συλλογή δεδομένων, rule-based engine, AI detection engine, risk scoring module και επίπεδο απόφασης. Κάθε συναλλαγή αξιολογείται σε milliseconds και λαμβάνει ένα ενοποιημένο risk score που καθορίζει αν θα επιτραπεί, επισημανθεί ή αποκλειστεί.

"Πώς μπορούμε να εντοπίσουμε αυτόματα επιθέσεις NFC relay σε ανέπαφα συστήματα πληρωμών, χρησιμοποιώντας συνδυασμό κανόνων και ML, χωρίς να επιβαρύνουμε την εμπειρία του χρήστη;"

Σύνδεση με Μαθήματα Σπουδών

  • Ασφάλεια Δικτύων & Επικοινωνιών — NFC πρωτόκολλα, threat modeling, relay attack mechanics
  • Μηχανική Μάθηση — Isolation Forest, Random Forest για anomaly detection και classification
  • Συστήματα Πληρωμών — EMV πρωτόκολλο, contactless payment flow, transaction latency
  • Ανάλυση Δεδομένων — Συνθετική παραγωγή dataset, feature engineering, αξιολόγηση μοντέλων

2. Πρόβλημα & Threat Model

Τα ανέπαφα συστήματα πληρωμών χρησιμοποιούν NFC (Near Field Communication) για γρήγορες συναλλαγές σε απόσταση έως 4cm. Ωστόσο, μια επίθεση NFC relay επιτρέπει σε έναν επιτιθέμενο να αναμεταδίδει την επικοινωνία μεταξύ μιας νόμιμης συσκευής και ενός τερματικού πληρωμών, εξαπατώντας το σύστημα ότι η κάρτα βρίσκεται κοντά, ενώ στην πραγματικότητα μπορεί να είναι χιλιόμετρα μακριά.

⚠ ΠΑΡΑΔΕΙΓΜΑ ΕΠΙΘΕΣΗΣ
Ο επιτιθέμενος A στέκεται κοντά στο θύμα (π.χ. σε μετρό). Ο επιτιθέμενος B στέκεται μπροστά σε ένα POS terminal. Η επικοινωνία μεταξύ κάρτας θύματος και POS αναμεταδίδεται wirelessly — η συναλλαγή εγκρίνεται χωρίς η κάρτα να είναι ποτέ κοντά στο terminal.

Γιατί τα Υπάρχοντα Συστήματα Αποτυγχάνουν

  • Rule-based μόνο: Εντοπίζουν γνωστά patterns αλλά αδυνατούν να ανιχνεύσουν εξελιγμένες ή νέες παραλλαγές επιθέσεων
  • AI-only μόνο: Ανιχνεύουν λεπτές ανωμαλίες αλλά με υψηλό false positive rate — αποκλείουν νόμιμες συναλλαγές
  • Έλλειψη real-time ανάλυσης: Τα περισσότερα συστήματα ανιχνεύουν απάτη μετά την ολοκλήρωση της συναλλαγής

3. Αρχιτεκτονική Συστήματος

Το σύστημα αποτελείται από πέντε επίπεδα που επεξεργάζονται κάθε συναλλαγή σε πραγματικό χρόνο:

  Contactless Payment Request (NFC)

       |             |

  [RULE-BASED    [AI DETECTION

Επίπεδο 1 — Data Collection Layer

Συλλέγει σε πραγματικό χρόνο τα χαρακτηριστικά κάθε συναλλαγής που είναι απαραίτητα για την ανίχνευση relay attacks:

  • Transaction Latency — καθυστέρηση round-trip σε milliseconds (relay attacks εισάγουν επιπλέον καθυστέρηση)
  • Transaction Amount — ποσό συναλλαγής (ασυνήθιστα μεγάλα ποσά σε νέο terminal)
  • Transaction Frequency — αριθμός συναλλαγών ανά χρονικό παράθυρο
  • Device State — κατάσταση συσκευής (screen on/off, app active, location services)
  • Contextual Info — γεωγραφική τοποθεσία, ώρα ημέρας, merchant category

Επίπεδο 2 — Rule-Based Engine

Ελέγχει για γνωστά, προκαθορισμένα patterns που υποδηλώνουν relay attack:

# Παράδειγμα κανόνων:
IF latency > 300ms          → SUSPICIOUS  (relay adds ~100-500ms delay)
IF transactions > 3/minute  → SUSPICIOUS  (rapid repeated attempts)
IF amount > €500 AND
   new_merchant = True       → SUSPICIOUS  (high value, unknown terminal)
IF device_screen = OFF AND
   transaction_initiated     → SUSPICIOUS  (card used without user action)

Επίπεδο 3 — AI Detection Engine

Δύο συμπληρωματικά ML μοντέλα για ανίχνευση ανωμαλιών και ταξινόμηση:

  • Isolation Forest — unsupervised anomaly detection, εντοπίζει στατιστικά ασυνήθιστες συναλλαγές χωρίς labeled data
  • Random Forest Classifier — supervised classification (legitimate / relay attack) με explainability μέσω feature importance
# AI Model Output:
{
  "isolation_forest_score": -0.42,   (negative = anomaly)
  "rf_prediction":          "relay_attack",
  "rf_confidence":          0.89,
  "top_features": [
    "latency_ms: 387 (threshold: 150)",
    "tx_frequency: 5 in 60s",
    "device_state: screen_off"
  ]
}

Επίπεδο 4 — Risk Scoring Module

Συνδυάζει τα αποτελέσματα των δύο engines σε ένα ενοποιημένο risk score (0–100) χρησιμοποιώντας weighted aggregation. Ο συνδυασμός μειώνει τα false positives του AI και τα blind spots των κανόνων.

Επίπεδο 5 — Decision Layer

  • Score < 30: ALLOW — η συναλλαγή προχωράει κανονικά
  • Score 30–70: FLAG — η συναλλαγή επισημαίνεται για manual review ή επιπλέον authentication
  • Score > 70: BLOCK — η συναλλαγή αποκλείεται αυτόματα και αποστέλλεται alert

6. ML Μοντέλα — Σημαντικές Λεπτομέρειες

Isolation Forest — Anomaly Detection

Ο Isolation Forest εντοπίζει anomalies χωρίς labeled data (unsupervised). Χτίζει τυχαία decision trees και μετρά πόσα splits χρειάζονται για να απομονωθεί ένα sample. Τα relay attacks δημιουργούν μοναδικά patterns (π.χ. ασυνήθιστο latency) που απομονώνονται γρήγορα — άρα λαμβάνουν χαμηλό anomaly score.

  • Ιδανικό για αρχικό φιλτράρισμα χωρίς labeled examples
  • Robust σε class imbalance (relay attacks είναι σπάνια)
  • Γρήγορο inference — κατάλληλο για real-time ανάλυση

Random Forest Classifier — Classification

Εκπαιδεύεται σε labeled συνθετικό dataset για να διακρίνει νόμιμες συναλλαγές από relay attacks. Παρέχει feature importance scores που εξηγούν γιατί μια συναλλαγή θεωρείται ύποπτη.

  • Ensemble μέθοδος — συνδυάζει πολλά decision trees για robustness
  • Παρέχει probability scores αντί binary αποφάσεων
  • Built-in explainability μέσω feature importance (χρήσιμο για audit)

Μετρικές Αξιολόγησης

  • Accuracy & F1-Score — συνολική απόδοση ανίχνευσης
  • Precision & Recall — ισορροπία μεταξύ false positives και false negatives
  • False Positive Rate — κρίσιμο: αποκλεισμός νόμιμων συναλλαγών βλάπτει την εμπειρία χρήστη
  • Detection Latency — ο χρόνος απόφασης πρέπει να είναι <50ms για real-time χρήση
  • ROC-AUC Curve — συνολική ικανότητα διάκρισης σε διάφορα thresholds
